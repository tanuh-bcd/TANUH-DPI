 Quick Start




    
curl

    

        
TOKEN=$(curl -s -X POST https://dpi-dev.tanuh.ai/privacy-filter/api/demo-token \
  -H "Content-Type: application/json" \
  -d '{"name":"Your Name","email":"you@example.com"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

curl -X POST https://dpi-dev.tanuh.ai/privacy-filter/api/redact \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/document.pdf"

        
    






    
Python

    

        
import requests

resp = requests.post(
    "https://dpi-dev.tanuh.ai/privacy-filter/api/demo-token",
    json={"name": "Your Name", "email": "you@example.com"},
)
token = resp.json()["access_token"]

with open("document.pdf", "rb") as f:
    r = requests.post(
        "https://dpi-dev.tanuh.ai/privacy-filter/api/redact",
        headers={"Authorization": f"Bearer {token}"},
        files={"file": ("document.pdf", f, "application/pdf")},
        timeout=600,
    )
print(r.json()["entity_counts"])

 Privacy Filter Documentation

        
Technical overview of the Privacy Filter service architecture and capabilities.

    






    
Overview

    

        The Privacy Filter is an AI-powered document de-identification service that detects and redacts Personally Identifiable Information (PII) and Protected Health Information (PHI) from clinical and administrative documents. It uses a multi-layered detection approach combining transformer-based NER models with deterministic rule-based detectors.
    






    
Architecture

    
    
        
  • Primary Model: openai/privacy-filter (HuggingFace token-classification) — detects personal PII including names, dates, identification numbers
    • 
        
  • Secondary NER Model: dslim/bert-base-NER — detects organisations, locations, and named entities
    • 
        
  • Rule-Based Detectors: Deterministic regex patterns for tax IDs, IFSC codes, IBAN, SSN, EIN, Aadhaar numbers, emails, URLs, and labeled fields
    • 
        
  • OCR Engine: Scanned PDFs are OCR'd at 200 DPI before detection
    • 
        
  • Redaction: Detected spans are replaced with entity-type labels (e.g., [PERSON], [EMAIL]) in the output document
    •

Privacy & Data Handling

    
    
        
  • All processing happens on-premises — no data leaves the server
    • 
        
  • Uploaded files are deleted after processing
    • 
        
  • Visitor IPs are stored as one-way hashes only
    • 
        
  • No document content is logged or persisted
    • 
        
  • Compliant with DPDP Act requirements for data minimization
    •

VariableDescription
