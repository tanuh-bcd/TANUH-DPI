Python

    

        
import requests

resp = requests.post(
    "https://dpi-dev.tanuh.ai/privacy-filter/api/demo-token",
    json={"name": "Your Name", "email": "you@example.com"},
)
token = resp.json()["access_token"]

with open("document.pdf", "rb") as f:
    r = requests.post(
        "https://dpi-dev.tanuh.ai/privacy-filter/api/redact",
        headers={"Authorization": f"Bearer {token}"},
        files={"file": ("document.pdf", f, "application/pdf")},
        timeout=600,
    )
print(r.json()["entity_counts"])

Privacy Filter

                
AI-powered document de-identification service that detects and redacts Personally Identifiable Information (PII) and Protected Health Information (PHI) from clinical and administrative documents.

            


            
Key Capabilities

            

                
FeatureDescription

                
                    
AI-Powered DetectionTransformer-based NER models identify names, dates, IDs, and medical record numbers

                    
Rule-Based DetectorsRegex patterns for Aadhaar, PAN, IFSC, SSN, emails, phone numbers, and more

                    
Multi-Format SupportPDF, DOCX, images (PNG/JPG/TIFF), plain text, CSV, Markdown, and DICOM files

                    
OCR IntegrationScanned PDFs are automatically OCR'd at 200 DPI before PII detection

                    
On-Premises ProcessingAll data stays on the server — no external API calls, fully DPDP Act compliant

                    
Fast ProcessingTypically 10–30 seconds per document on CPU

                
            

        


        
        

            
How to Use

            
Follow these three steps to de-identify a document.


            

                

                    
1. Upload Document

                    
Navigate to the Privacy Filter tab and click "Choose document to redact...". Upload a PDF, DOCX, image, or text file. Maximum file size is 25 MB.

                

                

                    
2. Redact Document

                    
Click "Redact Document". The AI will scan for personal information using transformer models and regex rules. Processing takes 10–30 seconds.

                

                

                    
3. Review & Download

                    
Review detected entities in the results panel. The service replaces PII with type labels (e.g., [PERSON], [EMAIL], [AADHAAR]). Download the original and redacted versions.

                

            


            

                Accuracy Notice
                AI detections may be incomplete or incorrect. Please review every output before use in production. The system prioritizes recall (catching more PII) over precision.
            

        


        
        

            
Detection Pipeline

            
The Privacy Filter uses a three-layer detection pipeline to maximize PII coverage.


            
Layer 1 — Primary NER

            
The openai/privacy-filter HuggingFace token-classification model detects personal PII entities: names, dates of birth, ID numbers, addresses, phone numbers, and medical record numbers.


            
Layer 2 — Secondary NER

            
The dslim/bert-base-NER model provides additional coverage for organisations, locations, and named entities that the primary model may miss.


            
Layer 3 — Rule-Based Detectors

            
Deterministic regex patterns catch structured identifiers:

            

                
PatternExample

                
                    
Aadhaar Number1234 5678 9012

                    
PAN CardABCDE1234F

                    
IFSC CodeSBIN0001234

                    
SSN / Tax ID123-45-6789

                    
Email Addressuser@example.com

                    
Phone Number+91 98765 43210

                    
IBAN / EINInternational bank account numbers

                
            


            
Processing Flow

            

                
1. Upload & Format Detection
File type identified. Images and scanned PDFs are routed through OCR.

                
2. Text Extraction
Text extracted via native parsing (PDF/DOCX) or Tesseract OCR (images/scanned).

                
3. NER Inference
Both transformer models run in parallel on extracted text.

                
4. Rule Scan
Regex patterns scan for structured identifiers (Aadhaar, PAN, SSN, etc.).

                
5. Entity Merging
Overlapping detections from all layers merged and deduplicated.

                
6. Redacted Output
PII replaced with type labels. Redacted file generated in original format.

Privacy & Data Handling

            
The Privacy Filter is designed with data minimization as a core principle.


            
    
                
  • On-premises only — all processing happens locally, no data leaves the server
    • 
                
  • No storage — uploaded files are deleted immediately after processing
    • 
                
  • No logging — no document content is logged or persisted to disk
    • 
                
  • Anonymous tracking — visitor IPs stored as one-way hashes only
    • 
                
  • DPDP Act compliant — meets India's data protection requirements for data minimization
    • 
            


            

                No External API Calls
                Unlike cloud-based solutions, the Privacy Filter runs entirely on-premises. Your documents never leave your infrastructure. Both NER models run locally on CPU.
