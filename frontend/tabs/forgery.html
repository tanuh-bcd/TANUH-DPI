 Usage Examples




    

        
1. Request a token

        
Returns an access_token valid for 24 hours.

        

            
curl -X POST "https://dpi-dev.tanuh.ai/forgensic/api/token" \
  -H "Content-Type: application/json" \
  -d '{"name": "Your Name", "email": "you@example.com"}'

            
        

    

    

        
2. Upload a document

        
Send as multipart/form-data. Returns a job_id.

        

            
curl -X POST "https://dpi-dev.tanuh.ai/forgensic/jobs" \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/document.pdf" \
  -F "ocr_enabled=true"

            
        

    

    

        
3. Poll job status

        
Poll until status is "complete" or "error".

        

            
curl -H "Authorization: Bearer $TOKEN" \
  "https://dpi-dev.tanuh.ai/forgensic/jobs/JOB_ID"

# Response:
{
  "job_id": "abc123",
  "status": "processing",
  "progress": 0.42
}

            
        

    

    

        
4. Fetch results + preview

        
Results include preview URLs; fetch them with the same token.

        

            
curl -H "Authorization: Bearer $TOKEN" \
  "https://dpi-dev.tanuh.ai/forgensic/jobs/JOB_ID/results"

curl -L -H "Authorization: Bearer $TOKEN" \
  "https://dpi-dev.tanuh.ai/forgensic/jobs/JOB_ID/files/preview_document_page_1.jpg" \
  -o preview.png

            
        

    







    
Python: End-to-end flow

    

        
import time, requests

BASE = "https://dpi-dev.tanuh.ai/forgensic"

token = requests.post(
    f"{BASE}/api/token",
    json={"name": "Your Name", "email": "you@example.com"},
    timeout=20,
).json()["access_token"]

headers = {"Authorization": f"Bearer {token}"}

with open("document.pdf", "rb") as f:
    r = requests.post(
        f"{BASE}/jobs",
        files={"file": f},
        data={"ocr_enabled": "true"},
        headers=headers,
        timeout=60,
    )
job_id = r.json()["job_id"]

for _ in range(30):
    s = requests.get(f"{BASE}/jobs/{job_id}", headers=headers, timeout=30).json()
    if s["status"] in ("complete", "error"):
        break
    time.sleep(2)

results = requests.get(f"{BASE}/jobs/{job_id}/results", headers=headers, timeout=30).json()
print(results)

# Example: download the first preview image
first_page = results["pages"][0]
preview_url = first_page.get("preview_url") or first_page.get("image_url")
if preview_url:
    img = requests.get(f"{BASE}{preview_url}", headers=headers)
    open("preview.jpg", "wb").write(img.content)

 Privacy & Data Handling

    

        Uploads are written to a temporary job folder while processing and then cleaned up. Annotated preview files are cached in memory for the job and cleared on server restart. Jobs are also purged after the TTL window configured on the server.

 What is Forgensic?

    

        Forgensic is a classical computer-vision forensic analysis system developed at TANUH (IISc) for the AB PMJAY Auto-Adjudication Hackathon.
        It detects tampering, splicing, copy-paste, and manipulation in clinical and administrative documents — without any machine-learning inference.
        The entire pipeline is deterministic, explainable, and runs entirely on CPU.

 Computer Vision Detection Pipeline

    

        Every uploaded page passes sequentially through the following classical CV checks. The pipeline is configured by the PIPELINE_PRESET environment variable
        (super_loose, npv_focus, full).
    


    

        

            
1

            
Image Pre-processing & Page Rendering
PDF pages rendered to high-resolution JPG via PyMuPDF (fitz). Single image files are used directly. DPI scaling applied for accuracy.

        

        

            
2

            
Copy-Paste Detection (C1)
ORB keypoint matching with FLANN-based KNN. Block-level hashing for duplicate region detection. Covers both pixel-level and compressed re-saves.

        

        

            
3

            
Overwriting Detection (C2)
Edge density and stroke-width analysis within text line bounding boxes. Ink inconsistency scoring against surrounding line baseline.

        

        

            
4

            
Added Content Detection (C3)
Stamp and signature heuristics using HSV color saturation thresholding, contour shape analysis, and aspect-ratio filtering. Flags rubber-stamp marks and pasted image overlays.

        

        

            
5

            
Erasure Detection (C4)
Laplacian noise variance and gradient smoothness analysis in areas adjacent to text lines. Detects white-out, digital erasing, or paint-bucket fills.

        

        

            
6

            
Document Merge Detection (C5)
Divides the page into horizontal bands. Computes noise fingerprint (Laplacian variance) per band. A significant jump between header and body bands indicates documents from different sources were merged.

        

        

            
7

            
Watermark Removal Detection (C6)
FFT frequency domain analysis for residual watermark ghost patterns. Background uniformity scoring to detect pixel-level artifacts from removal tools.

        

        

            
8

            
Irregular Spacing Detection (C7)
Token gap analysis within lines using word-level bounding boxes from the OCR output. Statistical outlier detection on inter-token distances flags inserted or deleted words.

        

        

            
9

            
AI-Generated / AI-Edited Flags (C8, C9)
Combines texture uniformity (too smooth), font embedding entropy, and micro-print rendering anomalies to flag documents that appear fully or partially AI-generated.

        

        

            
10

            
Annotated Preview Generation
Renders detected bounding boxes on each page image using Pillow. Colour-coded by category. Saved as preview_{page}.jpg and served from Redis-tracked file map.

        

    


    

         Preset tuning: Set PIPELINE_PRESET=super_loose in production to maximise sensitivity (flag more suspects). Use npv_focus when targeting only NPV-relevant forgeries. Use full for strictest, lowest false-positive settings.
    







    
 Forgery Finding Categories

    
Every detected region is tagged with one of the following 10 category codes. Multiple categories can be assigned to a single page.

    

        
CodeLabelDetection MethodExample Scenario

        
            

                C1
                Copy-Paste
                ORB keypoint matching, block hash
                Doctor's signature copied from another document
            

            

                C2
                Overwriting
                Edge density, stroke analysis
                Date or amount overwritten with different ink
            

            

                C3
                Added Content
                HSV saturation, contour shape
                Rubber stamp or signature pasted onto document
            

            

                C4
                Erasure
                Laplacian noise variance
                Patient name white-out and overtyped
            

            

                C5
                Document Merge
                Noise fingerprint band analysis
                Header from Hospital A, body from Hospital B
            

            

                C6
                Watermark Removal
                FFT frequency residuals
                "CONFIDENTIAL" watermark digitally removed
            

            

                C7
                Irregular Spacing
                Inter-token gap statistics
                Extra space inserted to change a cost figure
            

            

                C8
                AI-Generated
                Texture uniformity + font entropy
                Entire document generated by AI tool
            

            

                C9
                AI-Edited
                Partial texture anomaly
                Individual paragraphs re-written with AI
            

            

                C10
                No Discrepancy
                Absence of all above signals
                Clean, unmodified document

 Deployment Guide


    
 Docker (Recommended — Linux/GCP VM)

    

        
# Clone and configure
git clone https://github.com/tanuh-bcd/TANUH-DPI.git && cd TANUH-DPI
cat <<EOF > .env
FORGENSIC_SECRET_KEY=your-secure-random-key
REDIS_URL=redis://redis:6379/0
OCR_ENABLED=true
PIPELINE_PRESET=super_loose
EOF

# Build and launch all services
docker-compose up -d --build

# Verify all containers are healthy
docker-compose ps
docker-compose logs -f forgensic-worker

        
    


    
 Bare-Metal (Windows — Development)

    

        
# 1. Start Redis (via WSL)
wsl -e bash -c "sudo service redis-server start"

# 2. Set environment variables (PowerShell)
$env:PYTHONPATH        = "C:\path\to\TANUH-DPI"
$env:REDIS_URL         = "redis://localhost:6379/0"
$env:DATA_DIR          = "$env:TEMP\forgensic_data"
$env:OCR_ENABLED       = "true"
$env:PIPELINE_PRESET   = "super_loose"
$env:FORGENSIC_SECRET_KEY = "dev-secret-local"

# 3. Start the API
uvicorn forgensic.app.main:app --host 0.0.0.0 --port 8004 --reload

# 4. Start the Celery Worker (new terminal, same env vars)
celery -A forgensic.celery_app worker --loglevel=info --pool=solo -Q forgensic -n worker1@%h

         Scaling tip: On Linux/Docker, Celery uses the prefork pool by default.
        Set --concurrency to the number of CPU cores for maximum throughput. Each worker process is isolated, avoiding all OpenCV thread-safety issues.
