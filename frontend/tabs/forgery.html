 Usage Examples




    

        
1. Request a token

        
Returns an access_token valid for 24 hours.

        

            
curl -X POST "https://dpi-dev.tanuh.ai/forgensic/api/token" \
  -H "Content-Type: application/json" \
  -d '{"name": "Your Name", "email": "you@example.com"}'

            
        

    

    

        
2. Upload a document

        
Send as multipart/form-data. Returns a job_id.

        

            
curl -X POST "https://dpi-dev.tanuh.ai/forgensic/jobs" \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/document.pdf" \
  -F "ocr_enabled=true"

            
        

    

    

        
3. Poll job status

        
Poll until status is "complete" or "error".

        

            
curl -H "Authorization: Bearer $TOKEN" \
  "https://dpi-dev.tanuh.ai/forgensic/jobs/JOB_ID"

# Response:
{
  "job_id": "abc123",
  "status": "processing",
  "progress": 0.42
}

            
        

    

    

        
4. Fetch results + preview

        
Results include preview URLs; fetch them with the same token.

        

            
curl -H "Authorization: Bearer $TOKEN" \
  "https://dpi-dev.tanuh.ai/forgensic/jobs/JOB_ID/results"

curl -L -H "Authorization: Bearer $TOKEN" \
  "https://dpi-dev.tanuh.ai/forgensic/jobs/JOB_ID/files/preview_document_page_1.jpg" \
  -o preview.png

            
        

    







    
Python: End-to-end flow

    

        
import time, requests

BASE = "https://dpi-dev.tanuh.ai/forgensic"

token = requests.post(
    f"{BASE}/api/token",
    json={"name": "Your Name", "email": "you@example.com"},
    timeout=20,
).json()["access_token"]

headers = {"Authorization": f"Bearer {token}"}

with open("document.pdf", "rb") as f:
    r = requests.post(
        f"{BASE}/jobs",
        files={"file": f},
        data={"ocr_enabled": "true"},
        headers=headers,
        timeout=60,
    )
job_id = r.json()["job_id"]

for _ in range(30):
    s = requests.get(f"{BASE}/jobs/{job_id}", headers=headers, timeout=30).json()
    if s["status"] in ("complete", "error"):
        break
    time.sleep(2)

results = requests.get(f"{BASE}/jobs/{job_id}/results", headers=headers, timeout=30).json()
print(results)

# Example: download the first preview image
first_page = results["pages"][0]
preview_url = first_page.get("preview_url") or first_page.get("image_url")
if preview_url:
    img = requests.get(f"{BASE}{preview_url}", headers=headers)
    open("preview.jpg", "wb").write(img.content)

 Privacy & Data Handling

    

        Uploads are written to a temporary job folder while processing and then cleaned up. Annotated preview files are cached in memory for the job and cleared on server restart. Jobs are also purged after the TTL window configured on the server.
    















    

        
 Forgery Detection Documentation

        
Technical overview of the Forgensic document forgery detection system.

    






    
Overview

    

        Forgensic is a document forgery detection system that uses classical computer vision and forensic analysis techniques to detect tampering, splicing, and manipulation in clinical and administrative documents. Unlike ML-based approaches, Forgensic uses deterministic image processing pipelines for explainable, consistent results with bounding-box overlays highlighting suspicious regions.
    






    
Detection Pipeline

    
The analysis runs classical CV checks on each page and outputs categories plus localized regions:

    
    
        
  • Copy-paste detection: duplicate region matching to find moved content
    • 
        
  • Overwrite detection: edge density and stroke inconsistencies within text lines
    • 
        
  • Added content detection: stamp and signature heuristics with color + shape cues
    • 
        
  • Erasure detection: low-noise gaps and unnatural smoothness in text lines
    • 
        
  • Document merge detection: noise fingerprint shifts across bands
    • 
        
  • Watermark removal detection: residual frequency and background artifacts
    • 
        
  • Irregular spacing: spacing anomalies in lines and token gaps
    • 
        
  • AI-generated indicators: category-level flags when detected
    •

Privacy & Data Handling

    
    
        
  • All processing happens locally — no external API calls
    • 
        
  • Uploaded files are deleted immediately after processing
    • 
        
  • Preview images are cached in-memory only (cleared on restart)
    • 
        
  • No document content is written to disk or any database
    • 
        
  • Maximum file size: 25 MB
