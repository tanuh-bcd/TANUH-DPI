Forgery Detection API

Submit documents, poll status, and retrieve forensic analysis results. Built with Python + FastAPI + OpenCV.

No authentication required — open access for hackathon evaluation.

Python FastAPI OpenCV

Endpoints

Method Path Description
POST/jobsUpload a document and create a processing job

Usage Examples

1. Upload a document

Send as multipart/form-data. Returns a job_id.

curl -X POST "http://localhost:8004/jobs" \
  -F "file=@/path/to/document.pdf" \
  -F "ocr_enabled=true"

2. Poll job status

Poll until status is "complete" or "error".

curl "http://localhost:8004/jobs/JOB_ID"

# Response:
{
  "job_id": "abc123",
  "status": "processing",
  "progress": 0.42
}

3. Fetch results

Returns full forgery findings with per-page categories and regions.

curl "http://localhost:8004/jobs/JOB_ID/results"

4. Preview image

Fetch rendered page with bounding boxes.

curl -L \
  "http://localhost:8004/jobs/JOB_ID/files/page_001.png" \
  -o preview.png

Python: End-to-end flow

import time, requests

BASE = "http://localhost:8004"

with open("document.pdf", "rb") as f:
    r = requests.post(f"{BASE}/jobs",
        files={"file": f},
        data={"ocr_enabled": "true"}, timeout=60)
job_id = r.json()["job_id"]

for _ in range(30):
    s = requests.get(f"{BASE}/jobs/{job_id}", timeout=30).json()
    if s["status"] in ("complete", "error"):
        break
    time.sleep(2)

results = requests.get(f"{BASE}/jobs/{job_id}/results", timeout=30)
print(results.json())

Forgery Detection Documentation

Technical overview of the Forgensic document forgery detection system.

Overview

Forgensic is a document forgery detection system that uses classical computer vision and forensic analysis techniques to detect tampering, splicing, and manipulation in clinical and administrative documents. Unlike ML-based approaches, Forgensic uses deterministic image processing pipelines for explainable, consistent results with bounding-box overlays highlighting suspicious regions.

Detection Pipeline

The analysis runs multiple forensic checks on each page:

  • Error Level Analysis (ELA): Detects JPEG re-compression artifacts that indicate edited regions
  • Noise Analysis: Identifies inconsistent noise patterns across the document
  • Copy-Move Detection: Finds duplicated regions within the same page
  • Splicing Detection: Detects content pasted from external sources
  • Metadata Analysis: Examines file metadata for tampering indicators
  • Edge Detection: Highlights unnatural boundaries around edited areas

Technology Stack

Python 3.10+ FastAPI OpenCV
  • No ML models required — pure classical CV pipeline
  • No GPU needed — runs on CPU only
  • Lightweight — minimal dependencies, fast startup
  • Async processing — ThreadPoolExecutor for non-blocking job handling
  • In-memory state — jobs stored in memory with TTL-based cleanup
