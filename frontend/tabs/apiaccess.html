API Access

Get separate developer tokens for each service. Tokens are valid for 24 hours.

For API Developers: Each service uses an independent JWT signed with its own secret key. Use the token for the specific API you are calling via curl / Postman.
UI Uploads: When using the Clinical Document or Insurance Policy tabs, tokens are fetched automatically — no manual action required.

 Endpoint Reference





    
 Clinical Document API — /pdf2abdm

    

        

            
                

                    Method
                    Path
                    Auth
                    Description
                

            
            
                
POST/pdf2abdm/api/token🔓 OpenIssue a 1-day demo JWT for the Clinical API

                
POST/pdf2abdm🔒 BearerSync: PDF → ABDM FHIR DocumentBundle

                
POST/pdf2abdm/submit🔒 BearerAsync: submit PDF, returns task_id (202 Accepted)

                
GET/pdf2abdm/task-status/{id}🔓 OpenPoll async task status (PENDING / SUCCESS / FAILURE)

                
GET/pdf2abdm/health🔓 OpenService liveness + model status

                
GET/pdf2abdm/model-health🔓 OpenAI model readiness check

 Forgery Detection API — /forgensic

    

        

            
                

                    Method
                    Path
                    Auth
                    Description
                

            
            
                
POST/forgensic/api/token🔓 OpenIssue a 1-day demo JWT for the Forgery API
